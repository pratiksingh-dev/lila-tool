# LILA BLACK — Player Journey Visualizer

A web tool for level designers to explore player behavior on LILA BLACK maps. See where players move, where fights break out, which areas get ignored, and how bots compare to real players.

**Live tool:** [your-netlify-url-here]

---

## Running locally

**Requirements:** Python 3.9+

```bash
# 1. Clone the repo
git clone https://github.com/your-username/lila-visualizer
cd lila-visualizer

# 2. Install dependencies
pip install pandas pyarrow

# 3. Run the data pipeline
python scripts/process_data.py --input ./player_data --output ./public/data

# 4. Serve locally
cd public
python -m http.server 8080
```

Open `http://localhost:8080` in your browser. No Node, no build step, no environment variables.

## Features

- Real minimap backgrounds for all 3 maps with exact coordinate projection
- All 8 event types rendered with distinct markers (Position, BotPosition, Kill, Killed, BotKill, BotKilled, Loot, KilledByStorm)
- Humans and bots visually separated — different colors, different path styles
- Filter by map, date, and match
- Timeline playback — scrub through a match or let it play at Slow / Normal / Fast / Ultra speed
- Works in aggregate mode too — press Play with "All matches" selected to watch cross-match event density build up over time
- Heatmap overlays for movement density, kill zones, death zones, and loot hotspots
- Zone coverage grid — 8×8 overlay showing visited zones (blue) vs. dead zones (red)
- Live Insights panel — updates as you change filters: kill concentration, map coverage, bot vs. human behavior, loot-vs-fight distance, dead zone alerts
- Hover tooltips on every event marker showing event type, player, match, time, and world coordinates

## Updating with new data

When new match data arrives, re-run the pipeline and redeploy:

```bash
python scripts/process_data.py --input ./player_data --output ./public/data
git add public/data
git commit -m "Update match data"
git push
```

Netlify picks up the push and redeploys automatically.

## Notes on the dataset

- Only Feb 14 data was available during development (partial day). The pipeline handles any number of days — Feb 10–13 will load automatically when those files are present.
- `Kill` and `Killed` (human-vs-human combat) do not appear in Feb 14 data but are fully handled in the frontend and will render with the complete dataset.
- See [ARCHITECTURE.md](./ARCHITECTURE.md) for full technical decisions and trade-offs.
- See [INSIGHTS.md](./INSIGHTS.md) for three data-backed findings about map design.
