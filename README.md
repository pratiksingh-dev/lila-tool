# LILA BLACK — Player Journey Visualizer

A web tool for level designers to explore player behavior on LILA BLACK maps. See where players move, where fights break out, which areas get ignored, and how bots compare to real players.

**Live tool:** https://lila-tool.vercel.app/
**Repository:** https://github.com/pratiksingh-dev/lila-tool

---

## Running locally

**Requirements:** Python 3.9+, no other dependencies.

```bash
# 1. Clone the repo
git clone https://github.com/pratiksingh-dev/lila-tool
cd lila-tool

# 2. Install Python dependencies
pip install pandas pyarrow

# 3. Run the data pipeline (point to your player_data folder)
python scripts/process_data.py --input ./player_data --output ./public/data

# 4. Serve locally
cd public
python -m http.server 8080
```

Open `http://localhost:8080`. No Node, no build step, no environment variables.

## Features

- Real minimap backgrounds for all 3 maps with exact coordinate projection from README formula
- All 8 event types with distinct markers — Position, BotPosition, Kill, Killed, BotKill, BotKilled, Loot, KilledByStorm
- Humans and bots visually separated — blue paths vs. pink paths, toggle independently
- Filter by map, date, and match — match dropdown shows K/D/Loot stats upfront
- Timeline playback — scrub or play at Slow / Normal / Fast / Ultra speed
- Aggregate playback — press Play with all matches selected to watch cross-match density build over time
- Heatmap overlays — movement density, kill zones, death zones, loot hotspots
- Zone coverage grid — 8×8 overlay, blue = visited, red = zero traffic
- Live Insights panel — kill concentration, dead zones, bot vs. human behavior, loot-fight distance, all updating in real time as you change filters
- Hover tooltips on every event marker — event type, player, match, time, world coordinates

## Updating with new data

When new match data arrives, re-run the pipeline and push:

```bash
python scripts/process_data.py --input ./player_data --output ./public/data
git add public/data
git commit -m "Update match data"
git push
```

Vercel auto-deploys on every push to main.

## Notes on the dataset

- Only Feb 14 data was available during development (partial day). The pipeline handles any number of days — Feb 10–13 load automatically when those files are present.
- `Kill` and `Killed` (human-vs-human combat) are absent from Feb 14 but fully handled in the frontend — they render correctly with the complete dataset.
- `heatmap.js` is loaded from `cdnjs.cloudflare.com` at runtime. The heatmap overlay requires an internet connection.

See [ARCHITECTURE.md](./ARCHITECTURE.md) for technical decisions and trade-offs.
See [INSIGHTS.md](./INSIGHTS.md) for three data-backed findings about map design.
